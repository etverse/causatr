# causatr

Unified causal effect estimation for methodological triangulation: g-computation
(parametric g-formula + ICE), IPW (via WeightIt), and matching (via MatchIt).
Part of the [etverse](https://github.com/etverse) ecosystem.

## S3 classes

- `causatr_fit` — returned by `causat()`. Holds model(s), data, method, and metadata.
- `causatr_result` — returned by `contrast()`. Holds estimates, SEs, CIs, vcov.
- `causatr_diag` — returned by `diagnose()`. Holds balance, positivity, weight info.
- `causatr_intervention` — returned by `static()`, `shift()`, etc.

## Two-step API

```r
# Step 1: fit
fit <- causat(data, outcome = "Y", treatment = "A", confounders = ~ L1 + L2,
              estimator = "gcomp",        # or "ipw" / "matching"
              model_fn = stats::glm)   # or mgcv::gam, etc.

# Step 2: contrast
result <- contrast(fit,
  interventions = list(a1 = static(1), a0 = static(0)),
  type = "difference",            # or "ratio" / "or"
  ci_method = "sandwich")         # or "bootstrap"
```

## Testing rules

- Use `expect_snapshot(error = TRUE)` for testing error conditions
- NEVER delete or mock away failing tests — fix the source code
- NEVER add complex mocking to make tests pass
- **Truth-based simulation tests are mandatory for new features.** Every supported (method × treatment × outcome × intervention × variance) combination MUST have a test that asserts the point estimate, SE, and CI against an analytical or external reference (lmtp::lmtp_tmle in R, or delicatessen in Python). Smoke tests that only check "runs without error" are accepted only for combinations where the closed-form truth is unavailable AND no external reference exists — and even then they should be marked as 🟡 in `FEATURE_COVERAGE_MATRIX.md`.
- **Update the feature coverage matrix.** When adding, removing, or changing a feature, update `FEATURE_COVERAGE_MATRIX.md` to reflect the new state in the same PR as the test changes. The matrix is the source of truth for what's tested at what fidelity; divergence between the matrix and the test files is a bug.
- **External reference cross-checks.** When the analytical truth is hard to derive (longitudinal MTPs, time-varying confounding, complex DGPs), validate against `lmtp::lmtp_tmle()` once during test design, then pin the expected value with the validation noted in a comment block above the test.

## Key design decisions

- **ICE over forward simulation** — fewer models, sandwich variance, faster
- **GLMs/GAMs over ML** — valid sandwich inference without debiasing
- **Three methods** — gcomp (outcome model), IPW (treatment model), matching. Agreement → confidence.
- **MatchIt as Imports; WeightIt as Suggests test oracle (Phase 4 onward)** — matching is delegated to MatchIt because it is genuinely hard. IPW we own end-to-end via a single self-contained density-ratio engine that handles static + shift + scale_by + threshold + dynamic + IPSI uniformly using the user's `model_fn` / `propensity_model_fn` for the treatment density model. WeightIt stays only as an independent reference for the static binary case in tests, never on the runtime path. Phase 3 still calls `WeightIt::glm_weightit()` at runtime; the migration happens in Phase 4.
- **`model_fn` parameter** — user passes the fitting function (glm, gam, etc.) rather than hardcoding
- **`estimator`, not `method`** — `causat()` / `causat_survival()` take `estimator = c("gcomp", "ipw", "matching")`. The name is deliberately NOT `method` so `...` can forward cleanly to `WeightIt::weightit(method = ...)` / `MatchIt::matchit(method = ...)` without a shadowing collision. This is why e.g. `causat(data, ..., estimator = "ipw", method = "cbps")` "just works". When adding a new wrapper around another package, preserve this invariant: never use `method =` as a top-level causat argument.
- **Sandwich default** — fast, valid for GLMs; bootstrap fallback for GAMs
- **Intervention functions, not formulas** — maximum flexibility for MTPs, dynamic rules, IPSI
- **`contrast()` as separate step** — fit once, contrast many interventions
- **ci_method: sandwich or bootstrap only** — delta method is applied internally for ratio/OR contrasts; not a separate ci_method
- **Single influence-function engine** — `variance_if()` (`R/variance_if.R`) is the one entry point for sandwich variance across g-comp, IPW, matching, and ICE. All four methods compute a per-individual IF as Channel 1 (direct covariate-sampling term) + Channel 2 (one `apply_model_correction()` call per nuisance model, preceded by a single `prepare_model_if()` per model) and aggregate via `vcov_from_if()`. Matching uses `cluster = subclass` for cluster-robust aggregation; IPW's combined MSM + propensity correction is delegated to `prepare_propensity_if()` + `apply_propensity_correction()`. See `vignettes/variance-theory.qmd` Sections 3–4 for the derivation and the `R/variance_if.R` roxygen header for the engine's internal layout.
- **`type` parameter** — `causat()` accepts `type = NULL` (default, auto-detected from `id`/`time`), `"point"`, or `"longitudinal"`. Explicit `type = "longitudinal"` requires `id` and `time`.
- **Phase 4: unified self-contained IPW** — replace the Phase 3 WeightIt wrapper with a single internal engine that handles **all** intervention types (static + shift + scale_by + threshold + dynamic + IPSI) via density-ratio weights and an explicit weighted MSM. Static binary is the simplest case and goes through the same path. The MSM stays explicit (not collapsed to weighted means) so non-saturated dose-response MSMs and Phase 8 effect-modification MSMs have a slot. **Estimand gating:** ATT / ATC are only defined for static interventions on a binary 0/1 treatment — `check_estimand_intervention_compat()` aborts any other combination at contrast time with error class `causatr_bad_estimand_intervention`. **IPSI shortcut:** the density ratio collapses to `(δ·A + (1-A)) / (δ·p + (1-p))`; the unified engine uses this closed form instead of evaluating a density at two transformed points. **Oracles:** WeightIt moves to Suggests and serves as the variance oracle for the static binary case; `lmtp::lmtp_ipw(learners_trt = "SL.glm", folds = 1)` is added to Suggests as an independent **point-estimate** oracle for shift / IPSI (its EIF-based SE is not directly comparable to the M-estimation sandwich, so it is not used as a variance oracle). See `PHASE_4_INTERVENTIONS_SELF_IPW.md`.

## Architecture notes

- The NHEFS dataset ships with 1746 rows (117 have NA education). The book uses a 1629-row pre-cleaned subset. `compute_contrast()` handles this by excluding rows with NA predictions from the target population.
- **Variance engine architecture (`R/variance_if.R`).** One dispatcher `variance_if(fit, ...)` routes to four method-specific branches that all share a small set of primitives: `bread_inv()` ((X'WX)^-1 for GLMs with a `rlang::warn()` on the `MASS::ginv()` fallback, `$Vp` for GAMs); `iv_design_matrix()` (counterfactual model matrix, GLM/GAM split); `prepare_model_if()` / `apply_model_correction()` (Channel 2 per model, split into a gradient-independent prep step and a per-intervention apply step so a single `p x p` bread solve serves all interventions); `correct_model()` (thin wrapper for single-gradient callers, kept for ICE); `prepare_propensity_if()` / `apply_propensity_correction()` (IPW Channel 2 dispatcher; Branch A via `sandwich::estfun(asympt = TRUE)` hoisted outside the intervention loop, Branch B deferred to Phase 4); `resolve_fit_idx()` (computes the model's row indices relative to `fit$data` and asserts the `na.action` invariant); `vcov_from_if()` (aggregates IFs to a k×k vcov via `crossprod`, with optional `cluster =` for matching's within-subclass sum-then-square); `variance_if_numeric()` (two-tier fallback — Tier 1 recovers the full IF via `sandwich::estfun + bread`, Tier 2 falls back to `V1 + J V_β J^T` where V1 is the full `crossprod(Ch1_mat) / n^2` so off-diagonal Channel-1 covariance survives, with a warning about the dropped IF cross-term).
- **G-comp / matching / IPW** prepare once per model (`prepare_model_if` / `prepare_propensity_if`) before the intervention `lapply`, then reuse the prep across interventions — O(1) bread solves instead of O(k). **Matching** works entirely on `fit$details$matched_data` (n = n_match, cluster-robust on `subclass`) per the IF scope from vignette Section 4.3. **ICE** iterates `correct_model()` once per model in a forward sensitivity recursion, consuming `$d` from step k to build step k+1's gradient; ICE's outer loop cycles models rather than interventions so prep hoisting does not apply. **IPW**'s Branch A uses WeightIt's `glm_weightit` + `sandwich::estfun(asympt = TRUE)` / `sandwich::bread()` to get per-individual adjusted IFs. Non-Mparts WeightIt methods (`gbm`, `super`, `bart`, `optweight`, `energy`, `npcbps`, `cfd`) trigger a `rlang::warn()` at fit time (not contrast time) because `glm_weightit` silently drops the propensity-uncertainty correction for them. **Phase 4 collapses Branch A entirely**: `fit_ipw()` will produce a plain weighted GLM MSM (not a `glm_weightit`), the dispatcher will route every IPW fit to `prepare_propensity_if_self_contained()`, and Branch A survives only inside the test oracle that compares the self-contained engine against `WeightIt::glm_weightit()` on the static binary case. See `PHASE_4_INTERVENTIONS_SELF_IPW.md` §7–8.
- For IPW and matching with saturated MSMs (Y ~ A), the predict-then-average approach in `compute_contrast()` gives the same result as reading off model coefficients (the Jacobian is trivial). This is correct but slightly redundant — kept for architectural consistency across methods. **Caveat:** the saturated MSM cannot carry an `A:modifier` interaction term, so `fit_ipw()` and `fit_matching()` call `check_confounders_no_treatment()` upfront and **hard-abort** when `confounders` contains such a term — silently dropping it would return a pooled ATE when the user asked for effect modification. This is the core Phase 8 issue (`PHASE_8_INTERACTIONS.md`). Point gcomp does NOT have this problem because its outcome model uses the full `confounders` RHS directly.
- The `model_fn` parameter (default `stats::glm`) allows users to plug in any fitting function with signature `function(formula, data, family, weights, ...)`.
- **ICE intervention application — current-time treatment only.** `ice_apply_intervention_long()` applies the intervention to the current treatment column **only** and DOES NOT recompute treatment lag columns from the intervened values. Lag columns hold OBSERVED A_{k-1}, A_{k-2}, ... at every backward step, matching the canonical Robins iterated conditional expectation algorithm and `lmtp::lmtp_tmle()`'s sequential-regression shortcut. A previous implementation recomputed lag columns from the intervened treatment, which double-counted shift/scale/threshold/dynamic interventions: a `shift(delta)` intervention on a 2-period continuous treatment returned 3*delta instead of the structural 2*delta. Verified against lmtp on the linear-Gaussian DGP in `test-simulation.R`.
- ICE g-computation (`R/ice.R`) defers model fitting to `contrast()` because the sequential outcome models are intervention-dependent. `fit_ice()` stores metadata; `ice_iterate()` runs backward iteration per intervention.
- ICE sandwich variance uses stacked estimating equations (Zivich et al. 2024) implemented via `variance_if_ice()` — a forward sensitivity recursion calling `correct_model()` once per model (no `geex` dependency). The J V_β Jᵀ shortcut does NOT work for ICE (ignores upstream model uncertainty, can underestimate SE by 40%+). Note: ICE models are fit *backward* in time (K → 0) inside `ice_iterate()`, but sensitivities propagate *forward* (0 → K) inside the variance loop — both are the back-substitution of the block-triangular bread of the stacked M-estimation system, in different orderings. See `variance-theory.qmd` Section 5.4.
- ICE bootstrap resamples individuals (all person-period rows together) via `boot::boot()` with `parallel`/`ncpus` support. `contrast()` passes these through.
- For binary outcomes, the first ICE model uses `binomial` (actual 0/1 response), but pseudo-outcome models use `quasibinomial` (fractional logistic for predicted probabilities in [0,1]).
- Matching with `estimand = "ATE"` auto-selects `method = "full"` for MatchIt (nearest-neighbor only supports ATT/ATC). User can override via `...`.
- Bootstrap wraps the entire replicate in `tryCatch` — failed replicates (e.g. factor level mismatches in bootstrap samples) return NA and are excluded from the variance estimate (standard bootstrap practice; Davison & Hinkley 1997). A warning is issued when any replicates fail.
- The `by` parameter in `contrast()` stratifies estimates by levels of a variable. Internally, it loops over levels, constructing a subset expression for each and delegating to the standard contrast machinery. Results include a `by` column in the estimates and contrasts data.tables. **`by` is a stratification mechanism, not a refit.** It averages the already-fitted model's predictions over each subset. For genuine *effect modification*, the underlying model must include an interaction term (e.g. `confounders = ~ L + sex + A:sex` for gcomp). Today only point gcomp recovers heterogeneous effects correctly under `by`; IPW / matching / ICE have partial or missing interaction support (see Phase 8).
- `causatr_result` stores `family` and `fit_type` from the fit, enabling context-aware printing, plotting (e.g. "risk difference" vs "mean difference"), and log-scale choice for ratio/OR contrasts.
- `plot.causatr_result()` uses the `forrest` package (Suggests) for publication-ready forest plots. Adapts reference line, log scale, and axis labels to the contrast type and outcome family.
- `tidy.causatr_result()` and `glance.causatr_result()` provide broom-compatible outputs via `generics` (Imports).
- `knit_print.causatr_result()` (`R/knit_print.R`) renders a `causatr_result` as a metadata header + two tinytable HTML tables (intervention means + contrasts) when the object is emitted from a knitr/Quarto chunk. It is registered in `R/zzz.R`'s `.onLoad()` hook — conditionally, only when `knitr` is installed — so both `knitr` and `tinytable` stay Suggests-only. If either is missing at render time, the method falls back to `knitr::normal_print()`, which in turn dispatches to `print.causatr_result()`. Vignettes use `code-fold: show` in their YAML so readers can fold the code chunk and still see the formatted result table; `format_tt(markdown = TRUE)` (requires `litedown`, also Suggests) is used where table cells carry LaTeX math so `\(...\)` reaches MathJax.
- `confint.causatr_result()` recomputes CIs from the stored SE and vcov, correctly respecting the `level` argument.
- Multivariate treatments (`treatment = c("A1", "A2")`) are supported for g-computation (point and longitudinal). IPW and matching block multivariate treatment with an informative error (planned for Phase 4 / Phase 7).
- IPW weight diagnostics (`compute_weight_summary()`) use `weightit$treat.type` to determine group labels: "treated"/"control" for binary, per-level for multinomial, "overall" for continuous.
- IPW supports binary, categorical (multinomial), and continuous treatments via WeightIt's generalized propensity score. Matching is **binary-only**: `MatchIt` itself rejects any non-binary treatment (verified against the upstream docs). `fit_matching()` intercepts this upstream with a clear causatr-side error pointing users to `gcomp` / `ipw`.
- **External weights** are validated up front in `causat()` and `causat_survival()` via `check_weights()`: NA, Inf / NaN, negative, non-numeric, and mis-sized vectors are rejected with specific errors. This prevents silent NaN estimates or cryptic downstream GLM aborts. Zero weights are allowed as a pass-through.
- **`na.action = na.exclude` is rejected** in `causat()` / `causat_survival()` via `check_dots_na_action()` (error class `causatr_bad_na_action`). Under `na.exclude`, `residuals(model, "working")` is padded with NAs to the original data length while `model.matrix(model)` drops NA rows; `prepare_model_if()` then builds `r_score` at the wrong length and `apply_model_correction()`'s `d_fit * r_score` silently recycles into a garbage correction vector (R only emits a "longer object length" warning). Only `na.omit` (default) and `na.fail` are accepted. 2026-04-15 fourth-round critical review.
- **ICE sandwich under non-uniform weights (2026-04-14 fix).** `variance_if_ice_one()` builds the step > 0 cascade gradient as `g_k = sum_j w_{k-1,j} d_{k-1,j} X^*_{k,j} mu_eta_{k,j}`. The `w_{k-1,j}` factor comes from `A_{k-1,k} = E[d s_{k-1,j}/d beta_k]`, which carries the prior weights from the (k-1)-th weighted fit. Dropping this factor (the pre-2026-04-14 behavior) silently underestimates the SE by ~2x under heterogeneous weights. Verified against `lmtp::lmtp_tmle` and a 300-replicate Monte Carlo. See `vignettes/variance-theory.qmd` Section 5.4 and the inline comment block in `R/variance_if.R:variance_if_ice_one`.
- **Effect modification across methods is the Phase 8 topic** (`PHASE_8_INTERACTIONS.md`). Current status: point gcomp handles `~ L + sex + A:sex` correctly. IPW and matching **hard-abort** on any `A:modifier` term in `confounders` via `check_confounders_no_treatment()` in `R/utils.R` — the saturated `Y ~ A` MSM has no slot for the interaction and silently dropping it would return a pooled ATE, so the fitter rejects the request upfront with a Phase-8 pointer. ICE handles current-period `A:sex` but not `lag1_A:sex` (heterogeneity compressed across periods).
